How to Add Your Data for Hosting on the Website

  • Reply to this thread or personally message me with your github username so that it can be added on github repo

    • Detailed steps

  1. Fork the repository https://github.com/kerzendorf-group/group-data
  2. Clone your fork to your local system through the terminal
    git@github.com:kerzendorf-group/group-data.git
  3. Add remote to upstream
    git remote add upstream git@github.com:kerzendorf-group/group-data.git
    (can copy this command directly)
  4. Create a branch then switch to your branch.
    git checkout -b branch_name_like_bea_info upstream
    (this creates and switches you to the branch automatically)
  5. Your forked repository, group-data, contains a directory named members. Navigate there and find your own directory or create one if it does not already exist.
  6. Inside your own directory's json folder, copy all the files from members/template_person/
  7. The following figure shows the structure of what the directories should look like. Please note that the 'jsons' folder contains multiple JSON files and be sure to have them all.
  8. Next, fill out the JSONs:.
    1. The "category" key inside basic_info.json should be one of the following:
      "Investigator", “Graduates”, “Undergraduates”, “Post Graduates”, “Post Doctorates” (spelling and case-sensitive).
    2. Please note "current_role": "Your_role" is for your more personal role in group.
      Example:
      "category": "Undergraduates",
      "current_role": "Professorial Assistant"
    3. Also in basic_info.json, if you are currently a part of the Kerzendorf group, keep "group_end_year": "Present". Otherwise, enter the year of leaving so that your name can be placed in the Alumni category
    4. In social_links.json, if you do not have any of the social media accounts, you may leave them as an empty string. See the figure below for reference
    5. In the following JSON files - awards.json, books.json, docs.json, education.json, experience.json, outreach.json, projects.json - you may customize them as per your needs. For example, if you have not written any books, then the value of the 'section' key should be left blank (empty string), as shown below. Otherwise, the part below section key of json doesn't need to be changed.
    6. You may also customize fields as per your wishes, as shown below.
    7. Ensure that after uploading the any media files, such as a profile picture or PDFs, you update those changes in your own JSON files in your directory as well, with the correct path pointing to the file location.
  9. Add and commit your changes with git
    Example:
    git add .
    git commit -m "a message like my_name's changes"
  10. Push to upstream/main and create a Pull Request.
    Example:
    git push upstream branch_name_like_bea_info
  11. Finally, create a Pull Request.
