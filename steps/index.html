How to Add Your Data for Hosting on the Website

  • Reply to this thread or personally message me with your github username so that it can be added on github repo

    • Detailed steps

  1. Fork the repository https://github.com/kerzendorf-group/group-data
  2. Clone it to your local system by running git@github.com:kerzendorf-group/group-data.git in the terminal.
  3. Create a new directory with your name (if it doesn't exists already) and inside your directory's json folder copy all json files from template_person/jsons and enter your own details in those json files.
  4. The cloned repository's folder 'group-data' contains a directory named 'members'.The cloned repository's folder 'group-data' contains a directory named 'members'.
  5. Navigate to the 'members' directory and create your own folder if it does not already exist.
  6. Follow the folder structure of the 'template_person' folder and replicate the same structure for your own folder (as shown in the figure below).
  7. Please note that the 'jsons' folder contains multiple JSON files. Copy and paste them into your own 'jsons' folder.
  8. Please note that the “category” key inside basic_data.json should be spelled same “Graduates”, “Undergraduates”, “Post Graduates”, “Post Doctorates” only (case-sensitive), in order to group people together. Please add "current_role": "Your_role" inside your JSON file as the value for your latest role in group.
  9. Enter all the required information as well.It is important to maintain correct capitalization for everything. The data is case-sensitive
  10. It is important to note that if you are currently a part of the Kerzendorf group, then the value "group_end_year": "Present" should be entered in 'basic_info.json'. Otherwise, enter the year of leaving so that your name can be placed in the Alumni category.
  11. Inside jsons for example'social_links.json', if you do not have any of the social media accounts, you may leave them as an empty string. For example, if you do not have a GitHub link, you may leave the value of GitHub blank (empty string). See the figure below for reference.
  12. In the following JSON files - 'awards.json', 'books.json', 'docs.json', 'education.json', 'experience.json', 'outreach.json', 'projects.json' - you may customize them as per your needs. For example, if you have not written any books, then the value of the 'section' key should be left blank (empty string), as shown below.
  13. You may also customize fields as per your wishes, as shown below.
  14. Ensure that after uploading the required files, such as a profile picture or PDFs, you update those changes in your own JSON files present in your directory as well, with the correct path pointing to the file location.
  15. Finally, create a Pull Request.
