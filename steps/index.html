How to Add Your Data for Hosting on the Website

  • Do not delete any key inside jsons. You may leave key's value as empty string. And for JSONs with section key, just leaving that empty will not display the information in it.

    • Detailed steps

  1. Fork the repository https://github.com/kerzendorf-lab/group-data. If you cant access the repo then please ask for access.
  2. If you haven't already, generate an SSH key pair on your local machine. You can follow these steps:
    1. Open your terminal.
    2. Run the following command to generate a new SSH key pair:
      ssh-keygen -t ed25519 -C "your_email@example.com"
      Replace "your_email@example.com" with the email address associated with your GitHub account.
    3. Press Enter when prompted to accept the default file location and name.
    4. Enter a secure passphrase (optional) and confirm it.
  3. Add your SSH key to your GitHub account:
    1. Copy the contents of the public key file (id_ed25519.pub) to your clipboard. You can do this by running:
      cat ~/.ssh/id_ed25519.pub | pbcopy
    2. Go to your GitHub account settings and navigate to the "SSH and GPG keys" section.
    3. Click on "New SSH key" or "Add SSH key".
    4. Give your key a descriptive title (e.g., "My MacBook Pro") and paste the contents of the public key file into the "Key" field.
    5. Click "Add SSH key" or "Save changes".
  4. After adding your SSH key, you can use SSH to interact with GitHub repositories instead of HTTPS. When cloning the repository, use the SSH URL instead of the HTTPS URL:
    git clone git@github.com:your-username/group-data.git
  5. Add your SSH key to your GitHub account:
    1. Copy the contents of the public key file (id_ed25519.pub) to your clipboard. You can do this by running:
      cat ~/.ssh/id_ed25519.pub | pbcopy
    2. Go to your GitHub account settings and navigate to the "SSH and GPG keys" section.
    3. Click on "New SSH key" or "Add SSH key".
    4. Give your key a descriptive title (e.g., "My MacBook Pro") and paste the contents of the public key file into the "Key" field.
    5. Click "Add SSH key" or "Save changes".
  6. After adding your SSH key, you can use SSH to interact with GitHub repositories instead of HTTPS. When cloning the repository, use the SSH URL instead of the HTTPS URL:
    git clone git@github.com:your-username/group-data.git
  7. Clone your fork to your local system through the terminal
  8. Run cd group-data
  9. Create a new branch Eg: git branch my-data
    You may use any branch name instead of my-data
  10. Switch to new branch git checkout my-data
  11. Your forked repository, group-data, contains a directory named members. Navigate there and find your own directory or create one if it does not already exist.
  12. Inside your own directory's json folder, copy all the files from members/template_person/
  13. The following figure shows the structure of what the directories should look like. Please note that the 'jsons' folder contains multiple JSON files and be sure to have them all.
  14. Next, fill out the JSONs.
    1. Ensure that the profile picture you upload is square
    2. Info.json contains all of your personal information. The value of the Id key should be "firstname_lastname".
    3. If you are a student, we would want you to fill the education.json file with your course details. Additionally, if you have any relevant experience, such as a role in the Kerzendorf group, please fill out the experiences.json file accordingly.
    4. There are some other optional JSON files you can fill out to display additional information on your individual member page:
      • awards.json - Details about any awards or honors you have received.
      • documents.json - Information about documents, such as certificates or publications, that you would like to showcase.
      • outreach.json - Details about any outreach activities or community involvement you have participated in.
      • posters.json - Information about any posters or presentations you have given.
      • projects.json - Details about personal or professional projects you have worked on.
      • publications.json - Information about any publications, such as papers or articles, that you have authored.
      • social_links.json - Details about your social media profiles or other online presences.
      You can fill out the desired JSON files as per your preference to display the corresponding information on your individual member page.
    5. Ensure that after uploading the any media files, such as a profile picture or PDFs, you update those changes in your own JSON files in your directory as well, with the correct path pointing to the file location.
  15. Add and commit your changes with git
    Example:
    git add .
    Check status of changed files
    git status
    Commit those changes
    git commit -m "a message like my_name's changes"
  16. Push to upstream/main and create a Pull Request.
    Example:
    git push -u origin my-data
  17. Finally, to create a Pull Request: either click on Compare & pull request or go to Pull requests tab on right of code and click on New pull request
    • Select kerzendorf-group/group-data in base repository
    • Select incoming changes in base
    • Select your cloned repository in head
    • Select newly created branch (Eg: my-data) inside compare
